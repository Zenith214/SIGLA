# Database Schema Optimization

This migration implements several optimizations to the SIGLA-2 database schema:

## 1. New ML-Related Tables

Added the following tables to support machine learning functionality:

- **ml_model**: Stores trained machine learning models with metadata
- **ml_prediction**: Stores predictions generated by ML models for barangays
- **action_grid_classification**: Stores action grid quadrant classifications for barangay services
- **ai_insight**: Stores AI-generated insights based on ML predictions
- **ai_recommendation**: Stores specific recommendations derived from insights

## 2. Performance Optimizations

### Added Indexes

Added indexes to frequently queried columns to improve query performance:

- Added indexes on foreign keys (e.g., `barangay_id`, `user_id`, etc.)
- Added indexes on status fields used for filtering
- Added indexes on timestamp fields used for sorting
- Added composite indexes for common query patterns

### Improved Data Types

Optimized data types for better storage efficiency and performance:

- Used `@db.VarChar(n)` with appropriate lengths for string fields
- Used `@db.Text` for longer text content
- Used `@db.Decimal(p,s)` with appropriate precision and scale for decimal values

### Added @updatedAt Triggers

Added `@updatedAt` attribute to all `updated_at` fields to automatically update timestamps when records are modified.

## 3. Relationship Optimizations

- Added explicit relationships between all related tables
- Ensured proper foreign key constraints
- Added cascade behaviors where appropriate

## How to Apply This Migration

### Using Prisma Migrate

```bash
npx prisma migrate dev --name add_ml_tables
```

### Manual Application

If you prefer to apply the migration manually:

```bash
psql -U your_username -d your_database -f ./prisma/migrations/20240709_add_ml_tables/migration.sql
```

## Verification

After applying the migration, verify that:

1. All new tables are created
2. All indexes are created
3. Existing data is preserved
4. Relationships work correctly

## Rollback

If you need to roll back this migration:

```sql
-- Drop new tables
DROP TABLE IF EXISTS ai_recommendation;
DROP TABLE IF EXISTS ai_insight;
DROP TABLE IF EXISTS action_grid_classification;
DROP TABLE IF EXISTS ml_prediction;
DROP TABLE IF EXISTS ml_model;

-- Drop new enums
DROP TYPE IF EXISTS model_type;
DROP TYPE IF EXISTS quadrant_type;
DROP TYPE IF EXISTS insight_type;
DROP TYPE IF EXISTS recommendation_type;
DROP TYPE IF EXISTS priority_level;
```

Note: This rollback script does not remove the added indexes on existing tables. To remove those, you would need to drop each index individually.
