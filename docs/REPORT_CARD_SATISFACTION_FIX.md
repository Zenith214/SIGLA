# Report Card Satisfaction Score Discrepancy Fix

## Problem
The satisfaction scores displayed in the Service Area Performance section of the report card page did not match the scores reported by Gemini AI in the Executive Summary.

**Example for Barangay 17:**
- **Gemini AI reported:** Safety: 51.1%, Financial: 55.4%, Disaster: 57.6%, Social: 56.7%, Environmental: 58.1%
- **Report Card showed:** Safety: 70.6%, Financial: 71.6%, Disaster: 72.1%, Social: 71.6%, Environmental: 72.1%

## Root Cause
The report card page was calling `/api/funnel-analysis?useML=true` which uses the **old calculation methodology** (counting questions and averaging) instead of the **new cascading funnel methodology** (awareness → availment → satisfaction).

The executive summary API correctly uses `calculateServiceFunnelMetrics` from `@/lib/funnel-calculations`, which implements the cascading funnel approach. However, the report card was using a different API endpoint that had its own calculation logic.

## Solution
Modified `src/app/reportcard/page.tsx` to:

1. **Call the ML API directly:** Changed from `/api/funnel-analysis?useML=true` to `/api/ml/funnel-analysis?barangayId=${barangayId}&cycleId=${cycleId}`

2. **Ensure active cycle is available:** Added dependency on `activeCycle` and `cycleLoading` to ensure the cycle ID is available before fetching analytics

3. **Better error handling:** Added logging and error messages when the ML API fails or when no active cycle is found

## Changes Made

### File: `src/app/reportcard/page.tsx`

**Before:**
```typescript
const funnelResponse = await fetch(`/api/funnel-analysis?barangayId=${barangayId}&useML=true`);
```

**After:**
```typescript
const cycleId = activeCycle?.cycle_id;
if (!cycleId) {
  console.warn('No active cycle found, skipping funnel analysis');
  setLoading(false);
  return;
}

const funnelResponse = await fetch(`/api/ml/funnel-analysis?barangayId=${barangayId}&cycleId=${cycleId}`);
```

Also updated the `useEffect` dependencies to include `activeCycle` and `cycleLoading`:
```typescript
}, [searchParams, activeCycle, cycleLoading]);
```

## Result
- The report card now displays the same satisfaction scores as Gemini AI
- Both use the correct cascading funnel methodology
- Scores are consistent across all parts of the application
- The ML-enhanced funnel analysis is properly utilized

- All new features should use `/api/ml/funnel-analysis` or call `calculateServiceFunnelMetrics` directly
- The cascading funnel methodology is the correct approach and should be used consistently across the application
