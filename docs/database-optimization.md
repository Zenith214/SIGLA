# Database Optimization Documentation

## Overview

This document outlines the database optimizations implemented in the SIGLA-2 project to improve performance, ensure data integrity, and support machine learning functionality.

## Optimization Summary

### 1. New ML-Related Tables

Added the following tables to support machine learning functionality:

| Table | Purpose |
|-------|--------|
| `ml_model` | Stores trained machine learning models with metadata |
| `ml_prediction` | Stores predictions generated by ML models for barangays |
| `action_grid_classification` | Stores action grid quadrant classifications for barangay services |
| `ai_insight` | Stores AI-generated insights based on ML predictions |
| `ai_recommendation` | Stores specific recommendations derived from insights |

### 2. Performance Optimizations

#### Indexing Strategy

Added strategic indexes to improve query performance:

- **Foreign Key Indexes**: Added indexes on all foreign key columns to speed up join operations
- **Status Field Indexes**: Added indexes on status fields used for filtering
- **Timestamp Indexes**: Added indexes on timestamp fields used for sorting
- **Composite Indexes**: Added composite indexes for common query patterns

#### Data Type Optimizations

Optimized data types for better storage efficiency and performance:

- **String Fields**: Used `@db.VarChar(n)` with appropriate lengths
- **Text Content**: Used `@db.Text` for longer text content
- **Numeric Fields**: Used `@db.Decimal(p,s)` with appropriate precision and scale

#### Automatic Timestamp Updates

Added `@updatedAt` attribute to all `updated_at` fields to automatically update timestamps when records are modified.

### 3. Relationship Optimizations

- Added explicit relationships between all related tables
- Ensured proper foreign key constraints
- Added cascade behaviors where appropriate

## Best Practices for Future Development

### Adding New Tables

When adding new tables to the schema:

1. Use appropriate data types for each column
2. Add indexes for columns used in WHERE clauses, JOIN conditions, or ORDER BY clauses
3. Define explicit relationships with other tables
4. Use `@updatedAt` for timestamp fields that should be automatically updated

### Query Optimization

When writing queries:

1. Use indexed columns in WHERE clauses when possible
2. Avoid using functions on indexed columns in WHERE clauses
3. Use appropriate JOIN types (INNER, LEFT, etc.)
4. Limit the number of rows returned when possible

### Monitoring Performance

Regularly monitor database performance:

1. Check query execution times
2. Identify slow queries
3. Analyze index usage
4. Monitor disk space usage

1. Drop the new tables:

   ```sql

   DROP TABLE IF EXISTS ml_prediction;
   ```
