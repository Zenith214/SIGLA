# SIGLA-2 Database Optimization

## Overview

This document provides information about the database optimization performed on the SIGLA-2 project. The optimization includes adding new ML-related tables, improving performance through indexing and data type optimization, and ensuring data integrity through proper relationships.

## Changes Made

### 1. New ML-Related Tables

Added the following tables to support machine learning functionality:

- **ml_model**: Stores trained machine learning models with metadata
- **ml_prediction**: Stores predictions generated by ML models for barangays
- **action_grid_classification**: Stores action grid quadrant classifications for barangay services
- **ai_insight**: Stores AI-generated insights based on ML predictions
- **ai_recommendation**: Stores specific recommendations derived from insights

### 2. Performance Optimizations

- **Added Indexes**: Added indexes to frequently queried columns to improve query performance
- **Optimized Data Types**: Used appropriate data types for better storage efficiency
- **Automatic Timestamp Updates**: Added `@updatedAt` attribute to all `updated_at` fields

### 3. Relationship Optimizations

- Added explicit relationships between all related tables
- Ensured proper foreign key constraints

## How to Apply the Changes

### Option 1: Using the Automated Script

```bash
node scripts/apply-db-optimizations.js
```

This script will:
1. Apply the database migrations
2. Generate the Prisma client
3. Verify the changes

### Option 2: Using Prisma Migrate

```bash
npx prisma migrate dev --name add_ml_tables
```

### Option 3: Manual Application

```bash
psql -U your_username -d your_database -f ./prisma/migrations/20240709_add_ml_tables/migration.sql
```

## Testing the Integration

To test the integration between the ML pipeline and the database, run:

```bash
node scripts/test-ml-db-integration.js
```

This script will:
1. Test the database connection
2. Verify that all ML tables exist
3. Test the ML Python integration with the database

## ML Pipeline Integration

The ML pipeline has been updated to integrate with the new database tables:

1. **Model Training**: Models are now saved to the `ml_model` table
2. **Prediction Storage**: Predictions are saved to the `ml_prediction` table
3. **Action Grid Classification**: Classifications are saved to the `action_grid_classification` table
4. **Insights and Recommendations**: Generated insights and recommendations are saved to the `ai_insight` and `ai_recommendation` tables

## Troubleshooting

If you encounter any issues with the database optimization:

1. Check the error messages in the console
2. Verify that the database connection is working
3. Ensure that you have the necessary permissions to create and modify tables
4. Check the Prisma schema for any syntax errors

If you need to roll back the changes, see the rollback instructions in the [Database Optimization Documentation](./docs/database-optimization.md).
